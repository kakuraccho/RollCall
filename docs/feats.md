# 実装する機能

## ユーザー登録
### 具体的な機能
- 一般ユーザーと管理者ユーザーの分類
    - `role`カラムをつけて管理
- 管理者ユーザーによる一般ユーザーの追加
    - 管理者が一般ユーザーの「仮パスワード」を発行
- 認証チェック
    - ginのミドルウェアを作成し、リクエストユーザーが「管理者かどうか」をjwtのペイロードから判別

### 必要な技術
- `user`テーブルの作成と`role`カラムで役職管理
- パスワードのハッシュ化 `crypto/bcrypto`
- jwtトークンの生成 `golang-jwt/jwt`

## 点呼担当
### 具体的な機能
- ローテーションに基づいて当番を決める
    - 日付や州番号から担当者を算出するロジック
    - `rotation`テーブルで基本ローテーションを管理
- 欠席申請に応じて入れ替える
    - 「特定の日付」と「代わりの担当者」を紐づける専用`assignments`テーブル
    - 基本ローテーションよりもこのテーブルを優先

### 必要な技術
- `rotation`,`assignments`テーブルの作成
- 日付操作ライブラリ 標準の`time`パッケージ
- 入れ替えロジックのユニットテスト
